!Работает только с 5 версией компилятора!

Более полную документацию см. http://wiki.dep111.rtc.local/knowledge:pvs-studio

В ветке alternative есть более полный пример
1. Установить PVS Studio
2. Установить переменную окружения PVS_STUDIO_PATH - она должна указывать на папку, в которой лежит CLMonitor.exe СО СЛЕШОМ НА КОНЦЕ (или пропишите этот путь в PATH)
3. Установить переменную окружения GIT_BASH_PATH - она должна указывать на папку, в которой лежит bash.exe СО СЛЕШОМ НА КОНЦЕ (скорее всего это C:\Program Files\Git\bin\)
   (или пропишите этот путь в PATH)
4. В Project->Options->User прописать:
        Before Compile      : .\scripts\_before_compile.bat #X #E
        
        Before Build/Rebuild: .\scripts\_before_build_dump.bat #X #P "Target 1"
        Здесь "Target 1" - имя вашего текущего таргета, должно быть в кавычках
        
        After Build         : .\scripts\_after_build.bat #X #P
        
Примеры этих скриптов есть в тестовом проекте.        
        
Действия при добавлении/удалении файлов в проект:
- Скрипт будет выдавать предупреждение вида `Project file was changed! Some files were added or removed:`
  (до тех пор, пока вы не закоммитите файл проекта)
- В Project->Options->User включить скрипты `Before Compile` и `Before Build`
- Перекомпилировать
- Снять галки обратно

Когда файлы не добавляются, можно держать все галки установленными, но это дает очень много шума в окне билда,
поэтому можно оставлять только галку After Build

Не забывайте для разных таргетов указывать имя соответствующего таргета при вызове скрипта Before Build!
